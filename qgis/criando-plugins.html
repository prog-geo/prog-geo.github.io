4.3. Construindo Plugins para o QGIS em Python¶

O QGIS fornece um plugin chamado Plugin Builder 3 que auxilia na preparação de um novo plugin Python através de um template. O Plugin Builder 3 pode ser instalado, a partir da janela principal do QGIS, acessando o menu Plugins | Manage and Install Plugins.... Na janela apresentada (Figura 4.19), selecione o nome do Plugin Builder 3 e pressione o botão de instalação (Install plugin).

Figura 4.19 - Instalação do Plugin Builder 3.¶

Nessa mesma janela, aproveite e faça a instalção do Plugin Reloader, como mostrado na Figura 4.20. Esse plugin possibilitará recarregar outros plugins em desenvolvimento quando realizarmos alterações em seus códigos.

Figura 4.20 - Instalação do Plugin Reloader.¶

Na janela Plugins | Settings, habilite também a apresentação de plugins experimentais (Figura 4.21), que será o caso do plugin que iremos criar nesta seção.

Figura 4.21 - Habilitando plugins experimentais.¶

Após a instalação desses dois plugins, você terá no menu Plugins as opções mostradas na Figura 4.22.

Figura 4.22 - Opções no menu Plugins: Plugin Builder 3 e Plugin Reloader.¶

4.3.1. Localização dos Plugins do QGIS no seu Sistema¶

Por padrão o QGIS mantém uma estrutura de pastas base onde armazena informações de configuração e o código de seus plugins. Em cada sistema operacional, temos uma localização específica, por exemplo:

No Microsoft Windows, esses arquivos encontram-se sob a pasta AppData\Roaming\QGIS\QGIS3, dentro da pasta principal do usuário, que, em geral, se localiza em C:\Users\%USERNAME% ou, apenas %HOMEPATH%.

No macOS, esses arquivos encontram-se sob a pasta Library/Application Support/QGIS/QGIS3, dentro da pasta principal do usuário, em geral, /Users/${USER}.

No Linux, o QGIS mantém os arquivos sob a pasta .local/share/QGIS/QGIS3, dentro da pasta principal do usuário, que, em geral, se localiza sob o endereço /home/${USER}.

Nota

No meu ambiente Linux Ubuntu 20.04 (Seção 4.1), esse caminho é o seguinte:

/home/gribeiro/.local/share/QGIS/QGIS3

O QGIS também permite descobrir essa localização através do console Python dentro do próprio QGIS. Acessando o menu Plugins | Python Console (Figura 4.23). Basta entrar com o seguinte comando:

>>> QStandardPaths.standardLocations(QStandardPaths.AppDataLocation)
['/home/gribeiro/.local/share/QGIS/QGIS3', '/usr/local/share/QGIS/QGIS3', '/usr/share/QGIS/QGIS3']

Figura 4.23 - Console do Python dentro da interface gráfica do QGIS.¶

Os plugins Python são mantidos na pasta profiles/default/python/plugins, sob a pasta principal de configuração do QGIS. Nesta pasta você encontrará os sub-diretórios pluginbuilder3 e plugin_reloader, que correspondem, respectivamente, às pastas dos plugins Plugin Builder 3 e Plugin Reloader, instalados no início desta seção.

Dica

No meu ambiente Linux Ubuntu 20.04 (Seção 4.1), esse caminho é o seguinte:

/home/gribeiro/.local/share/QGIS/QGIS3/profiles/default/python/plugins

Dica

Para descobrir onde o QGIS encontra-se instalado, utilize o comando abaixo no console Python dentro do próprio QGIS:

>>> QgsApplication.prefixPath()
'/usr'

4.3.2. Criando o Plugin Spectral¶

Vamos criar um plugin chamado Spectral. Para tal, abra o Plugin Builder, acessando o menu Plugins | Plugin Builder | Plugin Builder. Na janela apresentada (Figura 4.24), preencha as informações básicas sobre este novo plugin.

Figura 4.24 - Criando o plugin Spectral - Informações básicas do plugin.¶

Em seguida, forneça uma breve descrição desse plugin (Figura 4.25).

Figura 4.25 - Criando o plugin Spectral - Descrição do plugin.¶

Podemos escolher a criação do plugin baseado em alguns modelos (Figura 4.26). No caso, iremos selecionar a opção Tool button with dialog. Além disso, a opção de acesso à janela desse plugin será associada ao menu Plugins na interface principal do QGIS.

Figura 4.26 - Criando o plugin Spectral - Modelo do plugin.¶

Na janela seguinte (Figura 4.27), deixe todas as opções marcadas pois assim o modelo já dará suporte à internacionalização de código, isto é, você poderá fornecer arquivos com a tradução de textos e rótulos apresentados ao usuário.

Figura 4.27 - Criando o plugin Spectral - Opções de suporte no projeto do plugin.¶

A janela seguinte, permite que você inclua nos metadados do plugin, informações sobre o repositório de código fonte e o local para reportar bugs ou acessar documentação adicional de seu plugin (Figura 4.28).

Figura 4.28 - Criando o plugin Spectral - Metadados adicionais do plugin.¶

A última janela apresentada pelo Plugin Builder permite definir a pasta onde o código do plugin será gerado, a partir do modelo selecionado (Figura 4.29).

Figura 4.29 - Criando o plugin Spectral - Diretório alvo para geração do código do plugin.¶

Se tudo ocorrer bem, uma janela como a da Figura 4.30 será apresentada, com um resumo da crição do seu plugin.

Figura 4.30 - Criando o plugin Spectral - Sumário da criação do plugin.¶

4.3.3. Carregando o Plugin Spectral no QGIS¶

Reinicie o QGIS para que ele atualize a lista de plugins disponíveis no seu sistema. Ao abrir o QGIS novamente, selecione a opção de menu Plugins | Manage and Install Plugins.... Na janela que se abre, selecione o plugin com nome Spectral, como indicado na Figura 4.31

Figura 4.31 - Instalando o plugin Spectral.¶

Se o plugin for devidamente carregado, ao fechar a janela de instalação de plugins, você verá que um novo botão, como destacado na Figura 4.32 será apresentado, bem como a opção de menu Spectral será acrescentada ao menu Plugins.

Figura 4.32 - Botão e Menu de acesso ao plugin Spectral.¶

Se você tiver conseguido criar corretamente a estrutura do plugin, ao selecionar a opção Spectral no menu (Figura 4.32), uma janela como a da Figura 4.33 será apresentada.

Figura 4.33 - Janela do plugin Spectral logo após sua criação.¶

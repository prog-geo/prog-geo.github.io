6.4. Preparando o Ambiente para Inspeção do Código de um Plugin

Para inspecionar o código de um plugin Python durante sua execução no QGIS, precisamos utilizar uma ferramenta que permita ligar o processo do QGIS ao ambiente de depuração. Uma possível ferramenta a ser empregada para essa finalidade é o PyDev, que é uma IDE Python para o Eclipse. O PyDev permite realizar a depuração de um código Python de maneira remota (Remote Debugger). Uma maneira fácil de utilizar o PyDev é através da distribuição LiClipse, que já fornece uma versão pré-instalada do PyDev. Esta seção explica como utilizar o PyDev para inspecionar o código de um plugin Python do QGIS a partir do LiClipse.

Nota

Se você não tiver o QGIS ou o LiClipse instalados, considere a leitura da Seção 6.1.

6.4.1. Configurando o QGIS para uso do PyDev

Precisamos instruir o QGIS de maneira que a variável PYTHONPATH aponte para o diretório do arquivo pydevd.py distribuído com o LiClipse. Para localizar o arquivo pydevd.py, no Linux, utilize o seguinte comando:

find ~/liclipse/ -name pydevd.py

Esse comando irá produzir uma saída semelhante à apresentada abaixo:

/home/gribeiro/liclipse/plugins/org.python.pydev.core_7.5.0.202001101115/pysrc/pydevd.py

No QGIS, selecione o menu Settings | Options. Uma janela como a da Figura 6.34 será aberta.

Figura 6.34 - Opções do sistema para o QGIS.

Na opção System, seção Environment, acrescente a variável PYTHONPATH, associada ao caminho do módulo pydevd, como indicado na Figura 6.35.

Figura 6.35 - Configurando a variável PYTHONPATH no QGIS.

Nota

No meu sistema, usei o seguinte caminho:

/home/gribeiro/liclipse/plugins/org.python.pydev.core_7.5.0.202001101115/pysrc/

Após incluir essa variável, reinicie o QGIS. Em seguida, abra o console Python do QGIS e entre com o seguinte comando:

>>> import pydevd

A Figura 6.36 mostra a execução deste comando.

Figura 6.36 - Carregando o módulo pydevd para debug remoto no QGIS.

6.4.2. Criando uma Área de Trabalho no LiClipse

Ao executar o aplicativo LiClipse, será exibida uma janela como a da Figura 6.37.

Figura 6.37 - Criando uma área de trabalho (workspace) no LiClipse.

Nessa janela, indique o nome do diretório onde você deseja armazenar as configurações e projetos relativos ao desenvolvimento de plugins do QGIS. Esse diretório é chamado de workspace, no jargão do LiClipse. Você pode indicar um novo diretório, por exemplo, chamado QGIS-Plugins, que será automaticamente criado pelo LiClipse. Após confirmar essa pasta, uma janela como a da Figura 6.38 será apresentada, sem nenhum arquivo ou projeto ainda.

Figura 6.38 - Janela principal do LiClipse.

Nota

O LiClipse é baseado no Eclipse e, portanto, o conceito de workspace é o mesmo do Eclipse.

6.4.3. Configurando o Interperatdor Python no PyDev/LiClipse

Ao abrir o LiClipse, vamos configurar o interpretador Python. Para isso, acesse o menu Window | Preferences (Figura 6.39).

Figura 6.39 - Preferências do LiClipse.

Na janela que se abre, selecione a opção PyDev | Interpreters | Python Interpreter, como mostrado na Figura 6.40.

Figura 6.40 - Configurando o interpretador Python do PyDev/LiClipse.

Pressione o botão Browse for python/pypy exe para adicionar um interpretador. Uma janela como a da Figura 6.41 será apresentada. Forneça a localização do executável do interpretador Python bem como um nome fácil de referenciá-lo quando necessário.

Figura 6.41 - Definindo o nome e a localização do interpretador Python no PyDev/LiClipse.

A janela seguinte, Figura 6.42, apenas indica que o PyDev irá acrescentar alguns caminhos ao PATH do interpretador Python quando este for usado.

Figura 6.42 - Caminhos adicionados ao PATH do interpretador Python.

Se o executável do seu interpretador Python for corretamente informado, você verá a lista de pacotes disponíveis para uso na aba Packages (Figura 6.43). Nessa janela, apenas confirme as informações pressionando o botão Apply and Close.

Figura 6.43 - Configuração completa do interpretador Python no PyDev/LiClipse.

6.4.4. Configurando as Opções de Debug Remoto no PyDev/LiClipse

Agora, precisamos ativar as opções de Debug Remoto (Remote Debugger). Para que estas opções estejam disponíveis no menu PyDev da interface do LiClipse, selecione o menu Window | Perspective | Customize Perspective. Uma janela como a da Figura 6.44 será apresentada.

Figura 6.44 - Janela para ativação das opções de Debug Remoto na interface gráfica do PyDev/LiClipse.

Na aba Action Set Availability, selecione o item PyDev Debug como indicado na Figura 6.45. Em seguida, pressione o botão Apply and Close.

Figura 6.45 - Ativando os botões de ação de Debug Remoto para a interface gráfica do PyDev/LiClipse.

No menu PyDev, verifique se as opções mostradas na Figura 6.46 são apresentadas.

Figura 6.46 - Opções de Debug Remoto do PyDev.

6.4.5. Criando o Projeto do Plugin no LiClipse

Todas as configurações realizadas até o momento valem para a área de trabalho (workspace) criada, a qual se chama QGIS-Plugins. Vamos agora criar um projeto do PyDev. Acesse o menu File | New | Project.... Uma janela como a da Figura 6.47 será apresentada. Selecione a opção PyDev Project como indicado na figura e pressione o botão Next.

Figura 6.47 - Criando um projeto do PyDev.

Na janela seguinte (Figura 6.48), dê o nome do projeto de Spectral e marque a opção Create links to existing sources, para poder apontar para o diretório onde você instalou o plugin Spectral.

Figura 6.48 - Definindo o nome do projeto PyDev: Spectral.

Ao pressionar o botão Next, a janela da Figura 6.49 será apresentada. Aponte para a pasta onde o QGIS mantém os arquivos do seu plugin Spectral.

Figura 6.49 - Vinculando os arquivos do plugin Spectral ao projeto Spectral do PyDev.

Nota

A pasta do meu plugin Spectral encontra-se em:

/home/gribeiro/.local/share/QGIS/QGIS3/profiles/default/python/plugins/spectral/

Na próxima janela (Figura 6.50), pressione apenas o botão Finish.

Figura 6.50 - Finalizando a criação do projeto Spectral do PyDev.

Ao final, você deverá ter uma janela como a mostrada na Figura 6.51.

Figura 6.51 - Projeto Spectral do PyDev apontando para os arquivos do plugin Spectral do QGIS.

6.4.6. Inicializando o Debug Remoto

Para ativar o Debug Remoto, vá ao menu PyDev | Start Debug Server. Você verá uma mensagem na parte inferior do LiClipse, no Console, com a seguinte informação (Figura 6.52):

Debug Server at port: 5678

Figura 6.52 - Debug remoto inicializado na porta 5678.

Nota

Essa mensagem indica que o servidor foi inicializado e está aguardando uma conexão remota.

Antes de irmos à próxima seção, marque um ponto de parada (breakpoint) na linha 183 do arquivo spectral.py, como mostrado na Figura 6.53.

Figura 6.53 - Marcando um ponto de parada no projeto Spectral do PyDev/LiClipse.

Aproveite também para deixar o LiClipse na perspectiva de Debug. Para isso, acesse o menu Window | Perspective | Open Perspective | Other ... e selecione a opção indicada na Figura 6.54.

Figura 6.54 - Selecionando a perspectiva de Debug no LiClipse.

A seguinte janela deverá ser apresentada (Figura 6.55):

Figura 6.55 - Perspectiva de Debug no LiClipse.

6.4.7. Conectando o QGIS ao Debug do PyDev

No console Python do QGIS (Figura 6.56), entre com o comando para se conectar ao servidor de debug remoto:

>>> pydevd.settrace(port=5678, suspend=False)

Figura 6.56 - Conectando o QGIS ao debug remoto do PyDev/LiClipse.

A conexão pode levar alguns segundos para ser estabelecida. A Figura 6.57 mostra como fica a janela do LiClipse após o cliente, no caso, o QGIS, se conectar ao servidor de debug remoto.

Figura 6.57 - PyDev conectado para debug remoto ao QGIS.

Nota

Ao selecionar o plugin Spectral, na janela do QGIS (Plugins | spectral | Spectral), o LiClipse deverá parar na linha 188 do arquivo spectral.py, conforme mostrado na Figura 6.58.

Figura 6.58 - Parada do código do plugin na interface do LiClipse.

Deste ponto em diante, você poderá controlar a execução do seu plugin no LiClipse. Quando a linha de exibição da janela do plugin Spectral for executada, a janela mostrada na Figura 6.59 será apresentada.

Figura 6.59 - Janela do plugin Spectral.

Nota

Resumindo, o comando completo para uso no console Python do QGIS que estabelece a conexão ao servidor de debug remoto do PyDev pode ser realizado através do seguinte trecho de código:

import pydevd

try:
    pydevd.settrace(port=5678, suspend=False)
except:
    pass
